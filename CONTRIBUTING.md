# Участие в разработке

Спасибо за интерес к проекту. Ниже — минимальные правила, чтобы быстро включиться в разработку.

## Как внести вклад

1. **Fork** репозитория и клонируйте его к себе.
2. Создайте **ветку** под задачу: `git checkout -b feature/короткое-описание`.
3. Внесите изменения, при необходимости добавьте/обновите тесты.
4. Убедитесь, что код запускается локально (`python bot.py` с вашим `.env`).
5. Закоммитьте изменения: `git commit -m "Краткое описание"`.
6. Отправьте ветку в свой fork: `git push origin feature/короткое-описание`.
7. Откройте **Pull Request** в основной репозиторий.

## Ожидания к коду

- Соответствие существующему стилю (именование, отступы).
- Секреты и ключи — только в `.env`, не в коде и не в коммитах.
- Новые зависимости добавлять в `requirements.txt` с указанием версий или диапазонов.

## Структура и модули

- **`bot.py`** — логика бота (Telegram, OpenAI, вызовы Pinecone). Точка входа — `main()`.
- **`pine.py`** — обёртка над Pinecone: один класс `PineconeClient`, методы с docstring.
- **`data/data.txt`** — начальные факты для индекса; одна строка — одна фраза.

При изменении контрактов (например, формата метаданных в Pinecone) стоит обновить README и комментарии в коде.
